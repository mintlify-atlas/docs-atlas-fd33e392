## Understanding Document History

Automerge maintains a complete history of all changes made to a document. This allows you to:
- View the document at any point in its history
- Extract changes between specific points in time
- Implement undo/redo functionality
- Debug and audit changes

## Heads: Document State Markers

**Heads** are hash identifiers that represent a specific state of the document. Think of them as version markers or commit hashes in Git.

### Getting Current Heads

```typescript
import * as Automerge from "@automerge/automerge"

let doc = Automerge.from({ key1: "value1" })
const heads = Automerge.getHeads(doc)
console.log(heads) // Array of hash strings, e.g., ["abc123..."]
```

### Heads After Changes

Each change creates a new head:

```typescript
import * as Automerge from "@automerge/automerge"

let doc = Automerge.from({ count: 0 })
const heads1 = Automerge.getHeads(doc)

doc = Automerge.change(doc, d => { d.count = 1 })
const heads2 = Automerge.getHeads(doc)

doc = Automerge.change(doc, d => { d.count = 2 })
const heads3 = Automerge.getHeads(doc)

console.log(heads1 !== heads2) // true
console.log(heads2 !== heads3) // true
```

## Creating Views with view()

The `view()` function creates an immutable snapshot of the document at a specific set of heads.

<Note>
`view()` creates a **cheap reference-based copy** that shares memory with the original document. It's very fast, but the returned document is read-only.
</Note>

### Basic Time Travel

```typescript
import * as Automerge from "@automerge/automerge"

let doc = Automerge.from({ key1: "value1" })
const checkpoint = Automerge.getHeads(doc)

doc = Automerge.change(doc, d => {
  d.key2 = "value2"
})

doc = Automerge.change(doc, d => {
  d.key3 = "value3"
})

// View the document as it was at checkpoint
const past = Automerge.view(doc, checkpoint)
console.log(past) // { key1: "value1" }
console.log(doc)  // { key1: "value1", key2: "value2", key3: "value3" }
```

### Viewing and Extracting Changes

A powerful pattern is to view the past, then extract changes between two points:

```typescript
import * as Automerge from "@automerge/automerge"
import * as assert from "assert"

let doc = Automerge.from({ key1: "value1" })

// Maybe this is on another peer
let doc2 = Automerge.clone(doc)

const heads = Automerge.getHeads(doc)

doc = Automerge.change(doc, d => {
  d.key2 = "value2"
})

doc = Automerge.change(doc, d => {
  d.key3 = "value3"
})

// At this point we've generated two separate changes
// Now we want to send just those changes to someone else

// view() is a cheap reference-based copy at the given heads
const before = Automerge.view(doc, heads)

// This view doesn't show the last two changes
assert.deepEqual(before, { key1: "value1" })

// Get the changes to send to doc2
const changes = Automerge.getChanges(before, doc)

// Apply the changes at doc2
doc2 = Automerge.applyChanges(doc2, changes)[0]
assert.deepEqual(doc2, {
  key1: "value1",
  key2: "value2",
  key3: "value3"
})
```

## Working with Changes

### Getting Changes Between States

`getChanges()` returns the changes that occurred between two document states:

```typescript
import * as Automerge from "@automerge/automerge"

let doc = Automerge.from({ count: 0 })
const oldState = Automerge.clone(doc)

doc = Automerge.change(doc, d => { d.count = 1 })
doc = Automerge.change(doc, d => { d.count = 2 })
doc = Automerge.change(doc, d => { d.count = 3 })

const changes = Automerge.getChanges(oldState, doc)
console.log(changes.length) // 3 changes
```

### Getting All Changes

```typescript
import * as Automerge from "@automerge/automerge"

let doc = Automerge.from({ value: "initial" })
doc = Automerge.change(doc, d => { d.value = "updated" })
doc = Automerge.change(doc, d => { d.value = "final" })

const allChanges = Automerge.getAllChanges(doc)
console.log(allChanges.length) // 3 changes (including initial)
```

### Inspecting Individual Changes

```typescript
import * as Automerge from "@automerge/automerge"

let doc = Automerge.from({ message: "hello" })
doc = Automerge.change(doc, { message: "Update greeting" }, d => {
  d.message = "hello world"
})

const changes = Automerge.getAllChanges(doc)
const decoded = Automerge.decodeChange(changes[1])

console.log(decoded.message) // "Update greeting"
console.log(decoded.actor)   // The actor ID
console.log(decoded.time)    // Timestamp
```

## Using getHistory()

The `getHistory()` function provides a high-level view of the entire document history:

```typescript
import * as Automerge from "@automerge/automerge"

let doc = Automerge.from({ count: 0 })
doc = Automerge.change(doc, { message: "Increment" }, d => {
  d.count = 1
})
doc = Automerge.change(doc, { message: "Increment again" }, d => {
  d.count = 2
})

const history = Automerge.getHistory(doc)

history.forEach((state, index) => {
  console.log(`Change ${index}:`, state.change.message)
  console.log(`State:`, state.snapshot)
})

// Output:
// Change 0: undefined (initial state)
// State: { count: 0 }
// Change 1: Increment
// State: { count: 1 }
// Change 2: Increment again
// State: { count: 2 }
```

<Warning>
`getHistory()` is expensive for large documents as it reconstructs every state. Use it for debugging and auditing, not in performance-critical paths.
</Warning>

## Practical Use Cases

### Implementing Undo/Redo

```typescript
import * as Automerge from "@automerge/automerge"

class UndoableDocument {
  private doc: Automerge.Doc<any>
  private history: string[][] = []
  private historyIndex: number = -1

  constructor(initialState: any) {
    this.doc = Automerge.from(initialState)
    this.saveState()
  }

  private saveState() {
    const heads = Automerge.getHeads(this.doc)
    this.history = this.history.slice(0, this.historyIndex + 1)
    this.history.push(heads)
    this.historyIndex++
  }

  change(fn: (doc: any) => void, message?: string) {
    this.doc = Automerge.change(this.doc, { message }, fn)
    this.saveState()
    return this.doc
  }

  undo() {
    if (this.historyIndex > 0) {
      this.historyIndex--
      const heads = this.history[this.historyIndex]
      return Automerge.view(this.doc, heads)
    }
    return this.doc
  }

  redo() {
    if (this.historyIndex < this.history.length - 1) {
      this.historyIndex++
      const heads = this.history[this.historyIndex]
      return Automerge.view(this.doc, heads)
    }
    return this.doc
  }

  canUndo() { return this.historyIndex > 0 }
  canRedo() { return this.historyIndex < this.history.length - 1 }
}

// Usage
const undoDoc = new UndoableDocument({ count: 0 })
undoDoc.change(d => { d.count = 1 })
undoDoc.change(d => { d.count = 2 })

const current = undoDoc.undo() // Back to count: 1
const restored = undoDoc.redo() // Forward to count: 2
```

### Selective Sync

Send only recent changes to a peer:

```typescript
import * as Automerge from "@automerge/automerge"

let localDoc = Automerge.from({ users: [] })
let remoteDoc = Automerge.clone(localDoc)

// Track the last synced state
let lastSyncHeads = Automerge.getHeads(localDoc)

// Make some local changes
localDoc = Automerge.change(localDoc, d => {
  d.users.push({ name: "Alice" })
})
localDoc = Automerge.change(localDoc, d => {
  d.users.push({ name: "Bob" })
})

// Get only the changes since last sync
const sinceLast = Automerge.view(localDoc, lastSyncHeads)
const newChanges = Automerge.getChanges(sinceLast, localDoc)

// Send newChanges to remote peer
remoteDoc = Automerge.applyChanges(remoteDoc, newChanges)[0]

// Update sync marker
lastSyncHeads = Automerge.getHeads(localDoc)
```

### Diffing Between Points in Time

```typescript
import * as Automerge from "@automerge/automerge"

let doc = Automerge.from({ items: [] })
const v1 = Automerge.getHeads(doc)

doc = Automerge.change(doc, d => {
  d.items.push("apple")
  d.items.push("banana")
})
const v2 = Automerge.getHeads(doc)

doc = Automerge.change(doc, d => {
  d.items.push("cherry")
})
const v3 = Automerge.getHeads(doc)

// Get patches between any two versions
const patches_v1_to_v2 = Automerge.diff(doc, v1, v2)
const patches_v2_to_v3 = Automerge.diff(doc, v2, v3)
const patches_v1_to_v3 = Automerge.diff(doc, v1, v3)

console.log(patches_v1_to_v2.length) // Changes from v1 to v2
console.log(patches_v2_to_v3.length) // Changes from v2 to v3
console.log(patches_v1_to_v3.length) // All changes from v1 to v3
```

## Checking Head Validity

Verify if a document contains specific heads:

```typescript
import * as Automerge from "@automerge/automerge"

let doc = Automerge.from({ value: 1 })
const oldHeads = Automerge.getHeads(doc)

doc = Automerge.change(doc, d => { d.value = 2 })

// Check if heads are in the document's history
console.log(Automerge.hasHeads(doc, oldHeads)) // true

const fakeHeads = ["nonexistent-hash"]
console.log(Automerge.hasHeads(doc, fakeHeads)) // false
```

## Best Practices

<Tip>
1. **Store heads for important checkpoints** - Save heads when users create bookmarks, tags, or versions
2. **Use view() for read-only access** - It's much cheaper than cloning
3. **Batch change extraction** - Use `getChanges()` to efficiently sync ranges of changes
4. **Add descriptive messages** - Use the `message` option in `change()` for better history inspection
5. **Clean up old heads** - Don't store unbounded head history in memory
</Tip>

### Adding Meaningful Messages

```typescript
import * as Automerge from "@automerge/automerge"

let doc = Automerge.from({ tasks: [] })

doc = Automerge.change(
  doc,
  { message: "Add initial tasks" },
  d => {
    d.tasks.push({ title: "Write docs", done: false })
    d.tasks.push({ title: "Review PR", done: false })
  }
)

doc = Automerge.change(
  doc,
  { message: "Complete first task" },
  d => {
    d.tasks[0].done = true
  }
)

const history = Automerge.getHistory(doc)
history.forEach(h => {
  console.log(h.change.message)
})
// Output:
// undefined (initial state)
// Add initial tasks
// Complete first task
```

## Error Handling

<Warning>
`view()` will throw an error if any of the hashes in `heads` are not in the document's history.
</Warning>

```typescript
import * as Automerge from "@automerge/automerge"

let doc = Automerge.from({ value: 1 })

try {
  const invalidHeads = ["invalid-hash-12345"]
  const viewed = Automerge.view(doc, invalidHeads) // Throws!
} catch (error) {
  console.error("Invalid heads:", error)
}

// Better: check first
const headsToView = ["some-hash"]
if (Automerge.hasHeads(doc, headsToView)) {
  const viewed = Automerge.view(doc, headsToView)
} else {
  console.error("Heads not found in document history")
}
```

## Performance Tips

1. **view() is cheap** - Create views liberally; they share memory with the original
2. **getHistory() is expensive** - Only use for debugging or when you truly need all states
3. **Store heads, not snapshots** - Heads are small; use `view()` to reconstruct when needed
4. **Limit history traversal** - Don't iterate through history on every render
5. **Use diff() for UI updates** - More efficient than reconstructing full states

```typescript
import * as Automerge from "@automerge/automerge"

// ✅ Efficient: store heads, create views on demand
const checkpoints = new Map<string, string[]>()
let doc = Automerge.from({ data: [] })

checkpoints.set("initial", Automerge.getHeads(doc))

// Later: reconstruct efficiently
const initialHeads = checkpoints.get("initial")!
const initialState = Automerge.view(doc, initialHeads)

// ❌ Inefficient: storing full document snapshots
const snapshots = new Map<string, any>()
snapshots.set("initial", Automerge.clone(doc)) // Expensive!
```
