## Major Releases

<Accordion title="Version 0.7.0 - Columnar Compression (Major Release)">

This is a major release introducing columnar compression at runtime, achieving approximately **100x lower steady-state memory usage**.

**Added:**

- `ReadDoc::iter` and `ReadDoc::iter_at` for iterating over the whole document
- `{AutoCommit, Automerge}::apply_changes_batch` which efficiently applies a batch of changes at once
- `{AutoCommit, Automerge}::bundle` which compacts a set of changes in a similar manner to `Automerge::save` but without needing to include every change (experimental API)
- `{AutoCommit}::is_empty()` to determine if there are any commits at all in a document
- `{Automerge, AutoCommit}::{get_changes_meta, get_change_meta_by_hash}` to return just the metadata for a change by hash
- `Transactable::update_spans` now takes an `UpdateSpansConfig` parameter
- `Change::actors` which returns the actor IDs in a change
- `MarkSet::non_deleted_marks` returns a wrapper type for comparing MarkSets without deleted marks
- `Stats::{num_actors, cargo_package_name, cargo_package_version, rustc_version}`

**Changed:**

<Warning>
These are breaking changes. See the [Migration Guide](/reference/migration-guide) for details.
</Warning>

- `Automerge::{get_last_local_change, get_change, get_change_by_hash}` returns a `Change` not an `&Change` (changes are decompressed on demand)
- `hydrate::Text::new` and `hydrate::Value::text` now take a `TextEncoding` argument rather than a `TextRepresentation` argument
- `hydrate::Value::{apply_patches, apply}` now takes a `TextEncoding` argument
- The `MapRangeItem::key` field is now a `Cow<'a, str>` not a `&'a str`
- `ListRangeItem` and `MapRangeItem` no longer have a `R: RangeBound` parameter
- The `{ListRangeItem, MapRangeItem}::value` field is now a `ValueRef` not a `Value`
- `Span::Text` now has named fields rather than being a tuple variant
- `Mark` no longer has a lifetime type parameter and there is no longer a `Mark::into_owned` method
- `Mark::data` has become `Mark::{name, value}`
- `MarkData` has been renamed to `OldMarkData` and should generally not be used

**Removed:**

- `{AutoCommit, Automerge}::visualise_optree` - the internal storage is no longer a b-tree
- The `{ListRangeItem, MapRangeItem}::id` field is gone
- The `TextRepresentation` type no longer exists
  - `AutoCommit::{get_text_rep, set_text_rep, with_text_rep}` have been removed
  - `Automerge::{transact_and_log_patches, transact_and_log_patches_with, current_state, diff}` no longer take a `TextRepresentation` argument
  - `PatchLog::new` no longer takes a `TextRepresentation` argument

**Fixed:**

- Many small bugs with the spans and updateSpans methods and patch generation

</Accordion>

<Accordion title="Version 0.6.x - Text Encoding and Cursors">

### Version 0.6.1

**Bug Fixes:**
- Fix a bug where `{Automerge, AutoCommit}::get_marks` would return removed marks as mark with value `ScalarValue::Null` rather than not returning them at all

### Version 0.6.0

**New Features:**
- Add the ability to set the text encoding used when calculating indices into text objects via the `Automerge::new_with_encoding` constructor

- Allow configuring how the cursor position is resolved when the original character which the cursor referenced has been deleted

<Accordion title="Version 0.5.x - Marks, Spans, and Rich Text">

### Version 0.5.12

**New Features:**
- Allow empty keys in maps
- Add `SyncState::has_our_changes` to indicate whether we think the other end has everything we have
- Add `ReadDoc::stats` to obtain basic statistics about a document
- Allow configuring character widths for wasm32 target via utf-16 indexing feature flag

### Version 0.5.11

**Bug Fixes:**
- Fixed a bug where actor IDs were written incorrectly to the save document format, making it impossible to load the document

### Version 0.5.10

**Rich Text Features:**

Primary feature: block markers in rich text

- `ReadDoc::{spans, spans_at}` which return spans of text grouped by marks and separated by block markers
- `Transactable::{split_block, join_block, update_block}` for creating, removing, and updating block markers
- `Transactable::update_spans` for updating all block markers and text in a text sequence at once

**Bug Fixes:**
- Fix a bug where marks set to not expand at the end still produced splice patches containing the mark
- Fix a bug where splicing into the end of an expanding mark didn't produce patches with marks when receiving changes remotely
- Fix a bug where "undeleted" objects were not emitted in patches when patching in reverse

### Version 0.5.9

**Bug Fixes:**
- Fix a bug introduced in 0.5.8 which caused an error when loading a saved document with empty commits

**Performance:**
- Improve performance when diffing documents which contain a large number of objects

### Version 0.5.8

**Bug Fixes:**
- Fix a bug where the logic to rollback a transaction on error could panic
- Fix a bug where marks were calculated incorrectly when viewing a document at particular heads
- Update the `LoadOptions::migrate_strings` logic to no-op if there are no strings to convert

### Version 0.5.7

**Dependencies:**
- Update itertools dependency to 0.12.0

**Bug Fixes:**
- Fix a bug in `Read::get_marks` which caused it to ignore any heads passed to it

### Version 0.5.6

**New Features:**
- Add `Transactable::update_text`, which calculates a diff between the current value of a text field and a new value and converts that diff into splice operations

### Version 0.5.5

**Bug Fixes:**
- Fix a sync protocol backwards compatibility issue which caused 0.5.4 peers to emit messages that older peers couldn't understand in some circumstances

### Version 0.5.4

**Performance:**
- Performance improvements when loading documents with large numbers of objects
- Modify the sync protocol to send the entire document in the first message if the other end has nothing (dramatically improves initial sync speed)
- This is a backwards compatible change

### Version 0.5.3

**Performance:**
- Numerous performance improvements

**New Features:**
- Add `ReadDoc::get_marks` to get the marks active at a particular index in a sequence

### Version 0.5.2

## Release Process

Automerge uses separate release processes for different packages:

### Rust Package (`automerge`)

1. Version bump in `automerge/Cargo.toml`
2. Create PR and merge to main
3. Tag as `rust/automerge@<version>`
4. Publish with `cargo publish`

### JavaScript Packages

1. Version bump in `javascript/package.json`
2. Create PR and merge to main
3. Tag as `js/automerge-<version>`
4. CI automatically publishes to NPM

<Note>
The NPM publish token has a 30-day expiry and needs periodic manual refresh.
</Note>

## Deprecation Policy

Automerge follows semantic versioning:

- **Major versions** (X.0.0) - Breaking changes allowed
- **Minor versions** (0.X.0) - New features, backwards compatible
- **Patch versions** (0.0.X) - Bug fixes only

<Warning>
APIs marked as "experimental" may change in minor versions. Check release notes carefully.
</Warning>
